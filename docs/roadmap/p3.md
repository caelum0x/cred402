# p3 — Credit-as-a-service ("Cred402 Inside")

> **Let any x402 marketplace or protocol query Cred402 for agent creditworthiness.**
> Cred402 becomes the credit oracle and reputation layer the rest of the x402
> economy builds on — not a destination app, but infrastructure.

## Why
The data moat (p2) only compounds into a *protocol* if others rely on it. The
endgame is that any x402 marketplace, data market, or agent platform checks a
Cred402 reputation/credit score the way fintechs check a credit bureau — and can
extend Cred402-backed credit to agents inside their own product.

## Scope
- **Credit oracle.** An on-chain, queryable credit/reputation read surface other
  contracts can call (score, limit, risk flags, dispute rate) — Casper-rooted,
  cheap to read.
- **Embeddable SDK.** `@cred402/credit` for third-party x402 platforms: `creditCheck(agent)`,
  `openCreditLine(agent, terms)`, `simulate(signals)`, `preApprovalOffer(agent)` —
  REST + MCP + on-chain, all backed by the same risk policy.
- **Public read APIs (already partly built).** Generalize the bureau analytics
  (`/v1/discovery`, `/v1/agents/:id/*`, `/v1/credit/simulate`, credit offers) into a
  documented, versioned, rate-limited public product with API keys.
- **Agent Passport as the portable profile.** A shareable, verifiable credit file an
  agent presents to any platform.

## Deliverables
- Credit-oracle contract/read interface + `@cred402/credit` SDK + docs.
- Hardened, versioned public API (auth, quotas, SLAs) over the bureau services.
- A reference integration: a sample third-party x402 marketplace that gates or
  extends credit via Cred402.

## Success metric
A **third-party** x402 platform makes a credit/reputation decision using Cred402
(score read or Cred402-backed credit line opened inside their product).

## Out of scope
Liquidity/mainnet (p4), satellite execution (p5).
