# What Cred402 does

Cred402 has three layers.

## 1. **Agent identity layer**

Each AI agent has an on-chain Casper identity.

Example agents:

```text
RWAInspector.agent
RiskScorer.agent
CollateralMonitor.agent
TreasuryRouter.agent
DisputeJudge.agent
```

Each agent has:

```text
agent_id
public_key
service_type
stake
total_jobs_completed
x402_revenue_history
accuracy_score
dispute_rate
credit_score
current_credit_line
```

This uses Casper’s agent-identity story directly. The AI Toolkit says account abstraction allows agents to operate with their own on-chain identity instead of depending on a human wallet. ([casper.network][1])

---

## 2. **x402 revenue and receipt layer**

Agents earn money by servicing RWAs.

Example:

A DeFi protocol has tokenized a warehouse inventory asset. Before lenders fund it, they need verification.

The protocol asks:

```text
GET /api/rwa/warehouse/verify?id=WH-2049
```

The RWA verification agent replies:

```text
402 Payment Required
X-Payment-Amount: 0.002 CSPR
X-Payment-Network: casper
```

The lending agent pays through x402, receives the report, and Cred402 stores the signed receipt hash on Casper.

Receipt object:

```text
receipt_id
payer_agent
seller_agent
service_type
amount
timestamp
rwa_reference_hash
result_hash
payment_proof_hash
dispute_window
status: pending / settled / disputed / finalized
```

Now the agent has verifiable revenue.

That revenue history becomes the basis for credit.

Casper’s AI Toolkit describes exactly this kind of HTTP-native x402 flow, where an agent hits a paid endpoint, receives a `402 Payment Required`, signs a payment proof, and gets data back after verification. ([casper.network][1]) The casper-eip-712 repo also describes structured, domain-separated authorizations that make agent-initiated payments auditable and verifiable by contracts. ([GitHub][2])

---

## 3. **DeFi credit layer**

Agents borrow from a DeFi pool against their verified x402 cash flow.

Why would agents need credit?

Because an autonomous agent may need to pay for:

```text
satellite imagery
weather data
market data
IoT sensor data
KYC/compliance checks
LLM inference
transaction fees
RWA registry lookups
```

before it gets paid by the protocol that hired it.

Cred402 gives the agent a small revolving credit line.

Example:

```text
Agent: RWAInspector.agent
30-day x402 revenue: 128 CSPR
Completed jobs: 412
Dispute rate: 1.7%
Accuracy score: 94/100
Stake: 50 CSPR

Approved credit line: 40 CSPR
Max drawdown: 8 CSPR per job
Interest rate: 11% APR equivalent
```

The DeFi pool earns yield by financing productive agent work.

That is a real Casper-native financial primitive.

This is a flywheel.

# MVP architecture

## Smart contracts

Build five contracts, but keep each one small.

### 1. `AgentRegistry`

Registers autonomous agents.

State:

```rust
Agent {
    agent_id: String,
    owner_public_key: PublicKey,
    agent_public_key: PublicKey,
    service_type: String,
    stake: U512,
    reputation_score: u64,
    credit_score: u64,
    active: bool
}
```

Methods:

```text
register_agent(agent_id, service_type, agent_public_key)
stake(agent_id, amount)
slash(agent_id, amount, reason_hash)
update_reputation(agent_id, delta, evidence_hash)
set_credit_score(agent_id, score)
```

### 2. `X402ReceiptRegistry`

Stores payment receipt commitments.

State:

```rust
Receipt {
    receipt_id: String,
    payer_agent: String,
    seller_agent: String,
    amount: U512,
    service_type: String,
    result_hash: String,
    payment_proof_hash: String,
    timestamp: u64,
    status: ReceiptStatus
}
```

Methods:

```text
record_receipt(...)
finalize_receipt(receipt_id)
dispute_receipt(receipt_id, dispute_hash)
```

### 3. `RWAEvidenceRegistry`

Stores evidence about real-world assets.

State:

```rust
Evidence {
    rwa_id: String,
    agent_id: String,
    evidence_type: String,
    evidence_hash: String,
    confidence: u8,
    timestamp: u64,
    linked_receipt_id: String
}
```

Methods:

```text
submit_evidence(rwa_id, evidence_hash, confidence, receipt_id)
verify_evidence(rwa_id, evidence_id)
```

### 4. `AgentCreditPool`

The DeFi pool.

State:

```rust
CreditLine {
    agent_id: String,
    max_credit: U512,
    drawn: U512,
    interest_rate_bps: u64,
    due_timestamp: u64,
    status: Active / Frozen / Defaulted
}
```

Methods:

```text
deposit_liquidity()
withdraw_liquidity()
open_credit_line(agent_id, max_credit, interest_rate)
draw(agent_id, amount)
repay(agent_id, amount)
liquidate_or_slash(agent_id)
```

### 5. `RiskPolicyManager`

This is where Casper’s upgradable-contract story matters.

The policy decides:

```text
credit_line = f(revenue_history, dispute_rate, stake, accuracy, service_type)
```

For the hackathon, make it simple:

```text
base_limit = 0.3 * last_30_day_revenue
stake_multiplier = min(2.0, 1 + stake / 100)
dispute_penalty = max(0.2, 1 - dispute_rate * 5)
accuracy_multiplier = accuracy_score / 100

credit_line = base_limit * stake_multiplier * dispute_penalty * accuracy_multiplier
```

Then show that the policy can be upgraded from `v1` to `v2`.

Casper’s AI page emphasizes upgradable contracts as important for autonomous systems because contracts can evolve without redeployment. ([casper.network][1])

# Agent system

You need the agents to do real actions, not just print recommendations.

## Agent 1: `RWARequestAgent`

Creates an RWA verification job.

Actions:

```text
register RWA
request evidence
set bounty/payment amount
monitor evidence submissions
```

## Agent 2: `EvidenceSellerAgent`

Runs a paid API endpoint.

Example endpoint:

```text
GET /verify/solar-output?rwa_id=SOLAR-A17
```

It returns `402 Payment Required`, receives payment proof, then returns a signed verification report.

Actions:

```text
quote price
request x402 payment
verify payment proof
generate evidence report
submit report hash to Casper
```

## Agent 3: `CreditAgent`

Underwrites the agent.

Actions:

```text
read x402 receipt history
calculate revenue velocity
calculate dispute rate
calculate credit score
open or adjust credit line
```

## Agent 4: `TreasuryAgent`

Uses DeFi pool funds.

Actions:

```text
deposit liquidity
fund credit draw
monitor repayment
freeze credit line if risk changes
```

## Agent 5: `WatchdogAgent`

This is the “judge pleaser.”

Actions:

```text
listen to Casper streaming events
detect missed repayment
detect disputed evidence
slash stake
downgrade reputation
emit alert
```

Casper’s AI Toolkit highlights streaming events for real-time monitoring of deploys, transfers, and contract calls, which fits this WatchdogAgent role very naturally. ([casper.network][1])
