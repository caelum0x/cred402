Yes. We can directly use them — not just study them.

The rule is:

> **Cred402 core must be original. The surrounding rails can be open-source dependencies, forks, Docker services, SDKs, UI kits, agent frameworks, and sample integrations.**

That keeps us fast while staying safe for the Casper buildathon, because the rules require “original and newly developed” code/content, plus a working Casper Testnet prototype with a transaction-producing on-chain component. 

# Direct-use strategy

Use open-source in four modes:

```text
1. Package dependency
   Installed through npm, cargo, pip, go modules.

2. Forked service
   Clone/fork into our org and modify directly.

3. Docker sidecar
   Run the OSS service beside Cred402.

4. Template/sample import
   Copy permissively licensed sample code into our repo, keep attribution, then adapt heavily.
```

What we **do not** do:

```text
Do not clone a whole project and rebrand it as Cred402.
Do not copy another buildathon submission.
Do not copy core protocol logic from elsewhere.
Do not hide third-party code.
Do not use AGPL/GPL components inside core unless we intentionally accept the license obligations.
```

```text
Cred402 Agent Passport
Cred402 Agent ID / CAID
Cred402 Universal Receipt Envelope
Cred402 RWA Evidence Graph
Cred402 Proof-of-Service Revenue model
Cred402 Reputation Engine
Cred402 Agent Credit Score
Cred402 Agent Credit Pool
Cred402 Dispute and Slashing logic
Cred402 WatchdogAgent behavior
Cred402 Solar RWA demo flow
Cred402 UI narrative
Cred402 all-chain Casper-root architecture
```

That is the difference between:

```text
“we glued together libraries”
```

and:

```text
“we built a new protocol using open-source rails”
```
