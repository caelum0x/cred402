> **Make it real, phase 1.** The protocol's canonical layer stops being a
> simulation: Cred402 now builds, signs, and submits **byte-exact Casper 2.0
> deploys** with `casper-js-sdk`, reads live state from a real Testnet node, and

> deterministic demo and the test suite never need a node or a key.

## The write path

`casper-js-sdk@5` (the Casper 2.0 transaction model) produces a genuine deploy:
a `DeployHeader` bound to the chain name + signer public key, a `StoredContractByHash`
session (or `ModuleBytes` for installation), `standardPayment`, then
`Deploy.makeDeploy(...)` в†’ `deploy.sign(privateKey)`. The deploy hash is a real
blake2b over the CL byte serialization and the approval is a real ed25519 (or
secp256k1) signature.

**Build + sign are fully offline.** Only `signAndSubmit` / `installContract`
touch the network, so `test/casper_sdk.test.ts` constructs and *self-validates*
real signed deploys (`deploy.validate() === true`) with an ephemeral key and no
node вЂ” proving the integration is real crypto, not a mock.

## Run it

```bash
# Reads only вЂ” hits the real Casper 2.0 Testnet node, no key needed:
npm run casper:health
#   вњ“ reachable вЂ” chain "casper-test", build 2.2.1-вЂ¦
#   вњ“ state root hash: 1284d433вЂ¦

# Dry run вЂ” prints the casper-client plan + writes deploys.testnet.json:
npm run deploy:testnet

# LIVE deploy вЂ” build WASM, fund a key, then:
cd contracts && cargo build --release --target wasm32-unknown-unknown && cd ..
export CRED402_CHAIN=testnet
export CRED402_NODE=https://node.testnet.casper.network/rpc
export CRED402_SECRET_KEY=/path/to/funded/secret_key.pem
npm run deploy:testnet     # installs each Odra contract, writes real hashes
```

A funded Testnet key comes from the faucet at <https://testnet.cspr.live/tools/faucet>.
See `.env.example` for every variable.

## Why the SDK is isolated

`casper-js-sdk` is CommonJS and heavy, so it is imported **only** in
`sdk_signer.ts` / `install.ts`, which are loaded lazily by `factory.ts` on the
`testnet` path. The default `sim` path вЂ” and therefore the whole 87-test suite
and the deterministic demo вЂ” never loads the SDK. Going live is a transport swap,
exactly as `docs/architecture.md` promised.

## Safety

- No secrets in source: the key is read from `CRED402_SECRET_KEY` (a PEM path),
  `.env` is gitignored, and missing config fails loud rather than degrading.
- Every constructed deploy is `validate()`d before submission.
- Sim remains default, so CI and the demo are deterministic and key-free.
